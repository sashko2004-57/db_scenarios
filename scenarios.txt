ID: CreateAccount
НАЗВА: Створити обліковий запис
УЧАСНИКИ: Користувач, Система
ПЕРЕДУМОВИ:
РЕЗУЛЬТАТ: Обліковий запис
ВИКЛЮЧНІ СИТУАЦІЇ: Мінімум 1 обов'язкове поле порожне - EmptyRequiredField
Неправильна адреса електронної пошти - InvalidEmail
Пароль не відповідає вимогам - WeakPassword
Адреса електронної пошти вже використовується в іншому обліковому записі - EmailAlreadyInUse
?Неправильний код підтвердження - InvalidVerificationCode
ОСНОВНИЙ СЦЕНАРІЙ: 1. Користувач натискає кнопку "Реєстрація".
2. Користувач вводить дані в поля.
3. Користувач натискає кнопку "Зареєструватися".(можливі винятки EmptyRequiredField, InvalidEmail, WeakPassword)
4. Система перевіряє наявність облікового запису з введеною адресою електронної пошти.(можливий виняток EmailAlreadyInUse)
?5. Система надсилає лист з кодом підтвердження на введену адресу електронної пошти.
?6. Користувач вводить код підтвердження в поле.(можливий виняток EmptyRequiredField)
?7. Користувач натискає кнопку "Підтвердити".
?8. Система перевіряє введений код підтвердження.(можливий виняток InvalidVerificationCode)
9. Система створює обліковий запис.
10. Система повідомляє користувачу про створення облікового запису.

ID: SignIn
НАЗВА: Увійти в обліковий запис
УЧАСНИКИ: Користувач, Система
ПЕРЕДУМОВИ: Наявність облікового запису
РЕЗУЛЬТАТ: Доступ до облікового запису
ВИКЛЮЧНІ СИТУАЦІЇ: Мінімум 1 обов'язкове поле порожне - EmptyRequiredField
Неправильна адреса електронної пошти - InvalidEmail
Облікового запису не існує - AccountDoNotExist
Неправильний пароль - InvalidPassword
ОСНОВНИЙ СЦЕНАРІЙ: 1. Користувач натискає кнопку "Вхід".
2. Користувач вводить дані в поля.
3. Користувач натискає кнопку "Увійти".(можливі винятки EmptyRequiredField, InvalidEmail)
4. Система перевіряє дані.(можливі винятки AccountDoNotExist, InvalidPassword)
5. Система надає користувачу доступ до облікового запису.
6. Система повідомляє користувачу про вхід в обліковий запис.

ID: FindData
НАЗВА: Знайти дані
УЧАСНИКИ: Користувач, Система
ПЕРЕДУМОВИ: Наявність доступу до облікового запису
РЕЗУЛЬТАТ: Знайдені дані
ВИКЛЮЧНІ СИТУАЦІЇ: Дані, які відповідають критеріям, відсутні або недоступні - DataDidNotFound
ОСНОВНИЙ СЦЕНАРІЙ: 1. Користувач натискає кнопку "Пошук".
2. Користувач вказує критерії пошуку.
3. Користувач натискає кнопку "Знайти".
4. Система здійснює пошук.(можливий виняток DataDidNotFound)
5. Система відправляє знайдені дані користувачу.

ID: DownloadData
НАЗВА: Завантажити дані
УЧАСНИКИ: Користувач, Система
ПЕРЕДУМОВИ: Знайдені дані
РЕЗУЛЬТАТ: Файл з даними
ВИКЛЮЧНІ СИТУАЦІЇ: Помилка створення файлу - CreateFileError
Помилка завантаження - DownloadError
ОСНОВНИЙ СЦЕНАРІЙ: 1. Користувач обирає дані, які потрібно завантажити.
2. Користувач натискає кнопку "Завантажити".
3. Користувач обирає формат файлу.(можливий виняток CreateFileError)
4. Система створює файл з даними.(можливий виняток DownloadError)
5. Система відправляє створений файл з даними на пристрій користувача.
6. Система повідомляє користувачу про завантаження файлу.

ID: SendFeedback
НАЗВА: Відправити відгук
УЧАСНИКИ: Користувач, Система
ПЕРЕДУМОВИ: Знайдені дані
РЕЗУЛЬТАТ: Відправлений відгук
ВИКЛЮЧНІ СИТУАЦІЇ: Поле порожне - EmptyField
ОСНОВНИЙ СЦЕНАРІЙ: 1. Користувач обирає дані, щодо яких потрібно залишити відгук.
2. Користувач натискає кнопку "Відгук".
3. Користувач вводить відгук у поле.
4. Користувач натискає кнопку "Відправити".(можливий виняток EmptyField)
5. Система отримує відправлений відгук.
6. Система повідомляє користувача про отримання відгуку.

ID: ChangeSourcesList
НАЗВА: Змінити список джерел даних
УЧАСНИКИ: Адміністратор, Система
ПЕРЕДУМОВИ: Немає
РЕЗУЛЬТАТ: Змінений список джерел даних
ВИКЛЮЧНІ СИТУАЦІЇ: Неправильний формат запису джерела - InvalidSourceFormat
ОСНОВНИЙ СЦЕНАРІЙ: 1. Адміністратор натискає кнопку "Змінити список джерел" в меню "Адміністратор".
2. Система відправляє список джерел даних.
3. Адміністратор змінює список джерел даних(додає/редагує/видаляє джерела).
4. Адміністратор натискає кнопку "Зберегти".(можливий виняток InvalidSourceFormat)
5. Система зберігає змінений список джерел даних.
6. Система повідомляє адміністратора про зміну списку джерел даних.

ID: EditData
НАЗВА: Редагувати дані
УЧАСНИКИ: Адміністратор, Система
ПЕРЕДУМОВИ: Знайдені дані
РЕЗУЛЬТАТ: Відредаговані дані
ВИКЛЮЧНІ СИТУАЦІЇ: Неправильний формат запису даних - InvalidDataFormat
Мінімум 1 обов'язкове поле порожне - EmptyRequiredField
ОСНОВНИЙ СЦЕНАРІЙ: 1. Адміністратор обирає дані, які потрібно відредагувати.
2. Адміністратор натискає кнопку "Редагувати".
3. Адміністратор редагує дані.
4. Адміністратор натискає кнопку "Зберегти".(можливий виняток InvalidDataFormat, EmptyRequiredField)
5. Система зберігає відредаговані дані.
6. Система повідомляє адміністратора про отримання відредагованих даних.
